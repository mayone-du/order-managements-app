受注実績管理アプリ

概要

フリーランスのイラストレーターの彼女が、今までメモに実績を手書きしていたので、管理できるアプリを作りました。

彼女のためだけに作ったので他の人にはほぼ100%需要がないとおもいますが、どうせならちゃんと使えるものが作りたかったので、作ってみました。

使用技術

  • Django 3.1.7
    • virtualenv
    • Pillow
  • 素のJavaScript

機能一覧

  • 新規登録（サインアップ）
  • ログイン / ログアウト
  • 受注実績のCRUD操作
  • 実績数、売上額、平均評価の表示
  • JavaScriptで料金全てを3桁で,区切り

直したいところ

  • フォームのバリデーション
  • UI

かかった時間など

3/7からDjangoの学習をはじめて、機能自体は3/20くらいに完成しました。

そこからデザインとかに無駄に時間がかかったので公開が遅れました（汗）

余談

デザイン、HTMLへの変数の渡し方、DBの操作で手こずった。

それ以外は超スピーディに実装できた。RailsやLaravelはさわったことないが、Djangoの手軽さに感動しました。
